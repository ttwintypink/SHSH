# Clan Event Manager

Добавлено в сборку:

- `/event create` — создаёт событие через форму.
- Форма ограничена по длине:
  - название: 3–80 символов;
  - описание: 5–700 символов;
  - время публикации/закрытия отметок: строго `ДД.ММ.ГГГГ ЧЧ:ММ`, например `28.04.2026 18:00`;
  - лимит участников: 1–999.
- Время публикации события и закрытия отметок считается по МСК (`Europe/Moscow`).
- Оформление embed в стиле Apollo, цвет левой полоски: `0xB58CFF`.
- При создании события всем людям на сервере, кроме ботов, выдаётся роль `1467625280687308950` — “ещё не отметился”.
- Кнопки:
  - ✅ выдаёт `1467625274148655329`;
  - ❌ выдаёт `1467625406508171364`;
  - ❔ выдаёт `1467625283694624768`.
- Перед выдачей новой роли бот снимает все временные event-роли, чтобы у пользователя был только один статус.
- Автопинг роли `1467625280687308950` за 24 / 12 / 6 / 3 / 1 час до публикации события.
- Автопинг удаляется через 1 час.
- После закрытия отметок бот очищает все временные event-роли.
- `/event clear_roles` — ручная очистка временных event-ролей.
- `/event list` — список активных событий.

Важно:

1. У бота должно быть право `Manage Roles / Управление ролями`.
2. Роль бота должна стоять выше всех четырёх event-ролей.
3. В Discord Developer Portal должен быть включён `SERVER MEMBERS INTENT`.
4. В этой версии временные роли рассчитаны на одно активное событие одновременно. Если одно событие уже активно, бот не даст создать второе, чтобы роли не конфликтовали.
